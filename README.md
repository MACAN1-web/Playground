# Рейтинг абитуриентов колледжа

MVP веб-приложения для публикации рейтингов из Excel-файлов.

Подробная документация по архитектуре, запуску, админке, базе данных и типовым проблемам находится в `docs/PROJECT_DOCUMENTATION.md`.

## Запуск

```bash
cp .env.example .env
npm install
docker compose up -d postgres
npm run db:migrate
npm run dev
```

Публичное приложение: `http://localhost:5173`  
Скрытая админка: `http://localhost:5173/admin`  
API: `http://localhost:3001`  
Пароль админки задаётся в `.env`.

Открывайте точный адрес `Local`, который Vite показывает после запуска. Если порт `5173` занят, адрес будет, например, `http://localhost:5174`.

## Формат Excel-файла

Поддерживаются `.xls` и `.xlsx`. Основной формат загрузки — единый реестр заявлений с колонками:

- `Специальность`
- `Форма обучения`
- `СНИЛС абитуриента`
- `Средний балл аттестата`
- `Статус заявления` — может быть в файле, но на сайте не отображается

Система автоматически группирует заявления по паре `Специальность + Форма обучения`, объединяет дубли одного СНИЛС внутри такого списка и сортирует рейтинг по среднему баллу по убыванию. Одинаковая специальность на очной и заочной форме создаёт два отдельных списка. Контактные, паспортные и остальные лишние персональные данные не сохраняются.

Перед публикацией файл проверяется. Старые рейтинги заменяются только внутри одной транзакции после успешной проверки нового файла.

## Оригиналы документов

В админке доступен внутренний поиск по ФИО, полному СНИЛС, коду специальности или части её названия. Например: `09.02.09`, `веб` или `информационные системы`. Отметка `Оригинал предоставлен`:

- у одного СНИЛС может быть только одна активная галочка оригинала;
- если поставить галочку в другой специальности, старая галочка этого СНИЛС снимется автоматически;
- сохраняется после повторной загрузки реестра;
- не отображается в публичном приложении;
- поднимает абитуриента выше абитуриентов без оригинала, внутри группы с оригиналом сортировка идёт по среднему баллу.

ФИО хранится только для защищённого поиска в админке и не возвращается публичными API.


## PostgreSQL и перенос данных

Для локальной разработки PostgreSQL запускается через `docker compose up -d postgres`. Структура базы создаётся командой:

```bash
npm run db:migrate
```

Чтобы один раз перенести текущие данные из `data/ratings.db` в PostgreSQL:

```bash
npm run db:transfer-sqlite
```

Команда полностью заменяет содержимое PostgreSQL данными из SQLite. Для другого файла задайте `SQLITE_PATH`.
